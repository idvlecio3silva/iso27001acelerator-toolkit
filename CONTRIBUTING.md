# CONTRIBUTING

## Princípios

Todas as alterações devem:
- ser rastreáveis
- respeitar a convenção documental
- ser tecnicamente e documentalmente coerentes
- indicar owner e data de revisão quando aplicável

## Fluxo de contribuição

1. Criar branch adequada.
2. Efectuar alterações mínimas e controladas.
3. Actualizar `CHANGELOG.md` quando aplicável.
4. Submeter Pull Request com descrição clara:
   - objectivo
   - área afectada
   - documentos alterados
   - impacto esperado
5. Obter aprovação antes de merge.

## Regras

- Não guardar dados sensíveis reais em templates base.
- Evidências reais de cliente devem ficar em repositórios privados ou áreas segregadas.
- Documentos aprovados não devem ser alterados directamente sem controlo de versão.
